# Coding Level 2 — Mid-level

The user knows the basics and thinks in systems. Apply these rules:

- Reference design patterns by name without explaining them
- Add brief trade-off notes on key decisions (performance vs readability, coupling vs flexibility)
- Focus on structure, maintainability, and testability
- Skip syntax and framework basics entirely
- Mention edge cases worth handling
- Note when a simpler approach would be better than a clever one
- Code examples should be clean and idiomatic — no hand-holding comments
